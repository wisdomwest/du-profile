Addition 3 — New tab: Collaboration Map
Co-authorship & collaboration network
Shows which Daystar researchers already collaborate, which clusters are siloed, and which external institutions Daystar has the strongest ties to. Data comes automatically from ORCID once the harvest is live — no extra collection needed.
Developer note: Data source is ORCID co-author names + affiliations from the works section — already harvested. Render as a D3.js force-directed graph. Daystar nodes are larger; external institution nodes are smaller. Edge weight = number of co-authored papers. Clicking a node shows a tooltip with researcher name, school, cluster, and paper count. Filter controls update the graph in real time.

Addition 4 — Shareable researcher profile page
Individual public profile — stable URL per researcher
Each faculty member gets a shareable page at a stable URL. The "View full profile" button on Faculty Cards opens it. External funders, consortium partners, and co-authors can be sent a direct link. Same data as the enriched card but laid out with full publication list, not just highlights.
Developer note: Route: /faculty/[slug] — e.g. /faculty/simwa-richard. Slug generated from faculty name. Add a "View full profile →" button to each card in the Faculty Cards grid. All data is already available from ORCID harvest + existing publication records — no new data needed. The contact buttons use the same mailto logic as the faculty cards.

Addition 6 — New tab: Team Builder
Consortium & grant team builder
Answers the question: "I want to build a multi-disciplinary team for a grant application — who across different schools has complementary expertise and overlapping SDG alignment?" Select clusters and SDGs, generate a suggested team instantly.
Developer note: All data comes from the enriched faculty records already on the platform — cluster tags, SDG tags, school. Query logic: for each selected cluster, find the highest-scoring active researcher (most publications + verified indexed count). Show SDG overlap across the suggested team. "Generate team" button reruns the query. Export sends a pre-filled email to DRICE with the team suggestion. No complex algorithm needed — simple ranked filter per cluster.
